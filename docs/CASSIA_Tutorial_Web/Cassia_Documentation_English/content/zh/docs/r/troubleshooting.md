这里我们列出了用户可能遇到的一些常见错误。如果您有任何其他问题，请随时在最后一节留下评论！

## 认证错误（错误401）

### 常见原因
* API密钥未正确加载
* 无效或过期的API密钥
* API信用额度不足

### 解决方案
```r
# 验证API密钥是否正确设置
key <- Sys.getenv("ANTHROPIC_API_KEY")
print(key)  # 应显示您的API密钥，而不是空字符串

# 如有必要，重置API密钥
setLLMApiKey("your_anthropic_api_key", provider = "anthropic", persist = TRUE)
```

确保：
* 验证您的平台账户上有足够的信用额度
* 如有必要，通过平台的计费门户添加资金

## 文件未找到错误

### 常见原因
* 文件路径不正确
* 缺少输入文件
* CASSIA自动文件匹配问题

### 解决方案
```r
# 验证文件路径是否正确且文件存在
file.exists("your_input_file.csv")  # 如果文件存在则返回TRUE
```

指定路径时：
* 必要时使用绝对路径：
  * 不正确：
  "data/input.csv"
  * 正确：
  "C:/Users/YourName/Project/data/input.csv"
* 确保输入和输出文件名符合CASSIA的预期：
  * 严格遵循命名约定
  * 检查文件扩展名是否匹配所需格式

## 权限错误

### 常见原因
* 文件已存在且被锁定/使用中
* 写入权限不足
* 目录访问限制

### 解决方案
```r
# 如果文件存在，在继续之前删除它
if(file.exists("output.csv")) {
    file.remove("output.csv")
}
```

其他步骤：
* 检查文件是否已在另一个程序中打开
* 关闭使用该文件的任何程序
* 等待几秒钟后重试
* 如有必要，使用不同的输出文件名

## 最佳实践
* 保持API密钥安全，切勿共享
* 在覆盖文件之前始终备份重要数据
* 在运行操作之前仔细检查文件路径和权限
* 维持足够的API信用额度以确保处理不中断
